Part I — Theoretical Questions
Concepts, definitions, and rules of abstract classes and interfaces
Q 01 What keyword is used to declare an abstract class in Java?
Ainterface
Babstract
Cextends
Dvirtual
✦ Explanation
The abstract keyword is placed before the class keyword to declare an abstract class: public abstract class ClassName. Java does not have a virtual keyword like C++.
Q 02 Which statement about abstract classes is TRUE?
AAn abstract class can be instantiated directly
BAn abstract class cannot be instantiated, but can be subclassed
CAn abstract class must contain only abstract methods
DAn abstract class cannot have constructors
✦ Explanation
An abstract class cannot be instantiated, but other classes can extend it. It may contain both abstract and non-abstract methods, and it can have constructors that subclasses call via super().
Q 03 What is an abstract method?
AA method that is private and cannot be inherited
BA method with a body declared in a superclass
CA method with no body that must be overridden in a subclass
DA method that belongs to an interface only
✦ Explanation
An abstract method has no body (only the method prototype/header) and must be overridden in any concrete subclass. If a subclass fails to override it, a compiler error results.
Q 04 What happens if a subclass does NOT override an abstract method from its abstract superclass?
AThe method is silently ignored
BA runtime exception is thrown
CA compiler error results
DThe subclass becomes final
✦ Explanation
If a concrete (non-abstract) subclass fails to override all abstract methods, a compiler error is produced. The subclass would need to also be declared abstract to avoid the error.
Q 05 An abstract class serves mainly as a _____ for other classes.
AInterface
BSuperclass (parent class)
CSubclass (child class)
DStatic utility class
✦ Explanation
An abstract class is designed to be a superclass. It represents the generic or abstract form of all classes derived from it, capturing common characteristics and behavior.
Q 06 Which of the following correctly describes an interface in Java?
AAn interface is a class that can be instantiated
BAn interface is similar to an abstract class with all abstract methods
CAn interface can contain method bodies
DAn interface uses the extends keyword in class definition
✦ Explanation
An interface is similar to an abstract class where all methods are abstract. It cannot be instantiated, and classes use the implements keyword to adopt an interface.
Q 07 What keyword does a class use to adopt an interface?
Aextends
Babstract
Cimplements
Dinherits
✦ Explanation
A class uses the implements keyword to adopt an interface: public class MyClass implements MyInterface. The class is then obligated to provide implementations for all interface methods.
Q 08 An interface is often compared to a _____ because a class must fulfill all its method requirements.
ABlueprint
BContract
CTemplate
DModule
✦ Explanation
An interface is like a contract. When a class implements an interface, it must adhere to the contract by providing implementations for all methods declared in the interface.
Q 09 How many interfaces can a Java class implement?
AOnly one
BOnly two
CMultiple interfaces
DNone — a class cannot implement any interface
✦ Explanation
Java allows a class to implement multiple interfaces. Interface names are separated by commas after the implements keyword: class X implements A, B, C. However, a class can extend only one superclass.
Q 10 What is the nature of fields declared inside a Java interface?
Aprivate and non-static
Bprotected and static
Cpublic, static, and final
Dabstract and mutable
✦ Explanation
All fields in an interface are implicitly public, static, and final. Because they are final, they must be initialized at the point of declaration.
Q 11 Which of the following is a valid abstract method declaration?
Apublic abstract void compute() { return; }
Bpublic abstract void compute();
Cprivate abstract void compute();
Dstatic abstract void compute();
✦ Explanation
A valid abstract method has no body and ends with a semicolon: public abstract void compute();. Abstract methods cannot be private (they must be overridden) or static.
Q 12 What is the correct syntax to declare an interface?
Apublic abstract class MyInterface { }
Bpublic interface MyInterface { }
Cpublic implements MyInterface { }
Dinterface abstract MyInterface { }
✦ Explanation
The correct syntax is public interface InterfaceName { method headers; }. The interface keyword replaces class.
Q 13 How many superclasses can a Java class extend simultaneously?
AUnlimited
BTwo
COnly one
DThree
✦ Explanation
Java supports single inheritance — a class can extend only one superclass. However, implementing multiple interfaces is allowed as a workaround for multiple inheritance.
Q 14 Which statement correctly differentiates an abstract class from an interface?
AAn interface can have non-abstract methods; an abstract class cannot
BAn abstract class can have both abstract and non-abstract methods; an interface (traditionally) has only abstract method headers
CBoth are identical in every way
DA class can implement multiple abstract classes
✦ Explanation
An abstract class can mix abstract and concrete methods, has constructors, and can have instance variables. A traditional interface contains only abstract method headers (and final static fields). A class also can extend only one abstract class but implement many interfaces.
Q 15 When a class implements multiple interfaces, what must it do?
AImplement methods from only the first interface
BProvide implementations for all methods specified in ALL of the interfaces
CChoose which interface's methods to implement
DNot implement any methods since interfaces are abstract
✦ Explanation
When a class implements multiple interfaces, it must provide concrete implementations for all methods declared in every interface it implements, or it must itself be declared abstract.
Q 16 Can you use an interface reference variable to point to an object of a class that implements that interface?
ANo, interface reference variables are not allowed
BYes, but only to call methods defined in the interface
CYes, and it can call any method from the implementing class
DNo, you must use the concrete class type
✦ Explanation
An interface reference variable can hold an object of any class that implements the interface. However, through that reference you can only call methods declared in the interface, not extra methods defined in the implementing class.
Q 17 Why must interface fields be assigned an initialization value?
ABecause they are abstract
BBecause they are automatically final
CBecause all interfaces must be instantiated at some point
DBecause the JVM requires it for memory allocation
✦ Explanation
Interface fields are implicitly final, meaning they are constants. Final variables must be given an initialization value at declaration time because they cannot be changed after that.
Q 18 Which of the following is the correct way to implement two interfaces in a class?
Apublic class X implements A implements B
Bpublic class X extends A, B
Cpublic class X implements A, B
Dpublic class X extends A implements B extends C
✦ Explanation
To implement multiple interfaces, list them separated by commas after a single implements keyword: public class X implements A, B.
Q 19 Abstract methods are used primarily to ensure that a subclass _____ the method.
Ahides
Bimplements
Cignores
Doverloads
✦ Explanation
Abstract methods are used to enforce that every concrete subclass provides its own implementation (override) of the method. This guarantees consistent behavior across all subclasses.
Q 20 If class SalariedEmployee extends abstract class Employee, what must SalariedEmployee do?
ADeclare itself as abstract too
BOverride all abstract methods declared in Employee
CCannot have its own fields
DImplement the interface of Employee
✦ Explanation
A concrete subclass must override all abstract methods of its abstract parent. In the lecture example, SalariedEmployee must override get_salary() from Employee.
Q 21 Which access modifier CANNOT be applied to an abstract method?
Apublic
Bprotected
Cprivate
Dpackage-private (default)
✦ Explanation
Abstract methods cannot be private because a private method cannot be overridden by subclasses. Since abstract methods must be overridden, making them private would create a logical contradiction.
Q 22 The @Override annotation before a method in a subclass means:
AThe method is new and not inherited
BThe method overrides a method from the superclass
CThe method is abstract
DThe method is static
✦ Explanation
@Override is an annotation that tells the compiler the method is overriding a method from its superclass or interface. The compiler will report an error if the method does not actually override anything.
Q 23 A class that implements an interface but does not implement all its methods must be declared as:
Afinal
Bstatic
Cabstract
Dinterface
✦ Explanation
If a class does not implement all methods of an interface, it must itself be declared abstract. The unimplemented methods are then abstract methods that a further subclass must implement.
